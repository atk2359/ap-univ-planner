# 愛知県立大学大学院 情報科学研究科 博士前期課程 履修シミュレーター (AP-Univ-Planner)

本ツールは、愛知県立大学大学院 情報科学研究科 博士前期課程における講義履修計画の策定を支援するための非公式Webアプリケーションです。
複雑な修了要件（専攻間振替や単位区分）を自動で計算し、2年間の履修計画を視覚的に管理できます。

## 🚀 公開URL
[https://ap-univ-tools-project.github.io/ap-univ-planner/](https://ap-univ-tools-project.github.io/ap-univ-planner/)

## ✨ 愛知県立大学大学院 情報科学研究科向けの機能
- **修了要件の自動判定**: 専門科目、専攻科目、関連科目、演習科目、特別研究の各区分に基づいた単位集計。
- **他専攻科目の振替計算**: 専攻科目への振替（上限4単位）や、関連科目への振替（上限2単位）を自動計算。
- **学期別時間割**: 愛知県立大学の学期区分（前期・後期）に合わせ、M1〜M2の計4学期を管理。
- **集中講義・特別研究対応**: 集中講義や特別研究単位も専用枠で管理可能。
- **カタログ拡張機能**: 標準データにない講義をユーザー自身がJSON形式で追加可能。

## 🛠 使い方
1. **所属コース（専攻）の選択**: 左上のメニューから自分の所属を選択してください。
2. **講義の登録**: カタログから講義名を選択し、開講学期・時限を指定して時間割に追加します。
3. **要件チェック**: 右パネルの「修了要件 進捗」を確認し、不足単位がないかチェックします。
4. **データの保存**: 「保存(.json)」から作成した計画をファイルとして保存できます。
   - ※ブラウザを閉じるとデータがリセットされるため、こまめな保存を推奨します。

## 📝 カタログのカスタマイズ

標準カタログにない最新の講義や、他学部の講義を追加したい場合は、以下の手順でシステムを拡張できます。

### 🛠️ 拡張手順
1.  **テンプレートの準備**: `templates/diff_template.json` をダウンロードします。
2.  **データの編集**: JSONファイル内の `coreCourses` や `majorMasters` に講義情報を追記します（詳細は下記のガイドを参照）。
3.  **インポート**: アプリ上の **「カタログ拡張」** ボタンから編集したファイルを読み込むと、選択肢に即座に反映されます。

### 📖 講義データ定義・編集ガイド

JSONファイルを編集する際は、以下の定義に従って記述してください。

#### 1. データ構造
* **`coreCourses`**: 全コース共通の科目データ（配列形式）。
* **`majorMasters`**: 各専攻ごとのデータ。内部でさらに「専門（`adv`）」と「関連（`rel`）」に分かれます。

#### 2. プロパティ定義
| プロパティ | 型 | 内容 | 備考 |
| :--- | :--- | :--- | :--- |
| `name` | string | **講義名** | 必須項目 |
| `schedule` | string | **表示用スケジュール** | 例: `"前期月2"`, `"開講なし"` |
| `sem` | string | **学期フラグ** | 前期: `"z"` / 後期: `"k"` |
| `day` | number | **曜日数値** | 月: `1` ～ 金: `5` |
| `period` | number | **時限数値** | `1` ～ `5` |
| `isIntensive` | bool | **集中講義フラグ** | `true` の場合、曜日・時限の指定は不要 |
| `isOther` | bool | **その他フラグ** | 共同研究等の場合 `true` |

#### ⚠️ 編集時の注意点
* **不定期・開講なしの扱い**: 曜日や時限が固定されていない、または「今年度は開講なし」の場合は、`day` や `period` プロパティを記述せず、`name` と `schedule` のみを記述してください。
* **JSON構文**: JSON形式のため、末尾のカンマ（`,`）の欠落や、中括弧 `{ }` の閉じ忘れにご注意ください。
* **データの更新**: 既に登録済みの講義と同じ名前で定義した場合、新しく読み込んだデータの内容で上書きされます。

## 🎓 メンテナンスと引き継ぎについて
本プロジェクトは、卒業等に伴う管理者の交代を前提としています。後任の管理者は以下の方法で運用を継続してください。

## 📂 ファイル構成
- `index.html`: アプリケーションのUI構造。
- `script.js`: 時間割制御、要件計算、JSON入出力等のメインロジック。

## 🔧 開発者向け：データの更新方法

ベースとなる講義データを更新する場合、`data.js` 内の `coreCourses` および `majorMasters` を直接編集してください。
編集後、`lastUpdated` 変数に日付を記述することで、画面上の「Data: YYYY-MM-DD」表示が更新されます。
